// 06 — projects

Production-grade work

Three end-to-end DevOps projects built from scratch — each one solving a real engineering problem with production-quality code, documentation, and architecture.

01

Crucible

Enterprise CI/CD pipeline — code commit to live deployment, automatically.

CI/CD DevSecOps ● Live

Every push to main triggers a fully automated pipeline: Maven compiles and tests the application, SonarQube enforces code quality and aborts if standards aren't met, Trivy scans for CVEs at both the filesystem and container image level, Nexus stores versioned artifacts, Docker builds and tags the image, and Tomcat deploys the application. Prometheus and Grafana provide real-time pipeline observability throughout.

Checkout→ Build→ Tests→ Quality Gate→ Trivy Scan→ Nexus→ Deploy
// key highlights
  • Pipeline aborts automatically if SonarQube quality gate fails — bad code never reaches production
  • Two-layer Trivy scanning — filesystem before build, container image after build
  • JaCoCo coverage reports fed directly into SonarQube quality gate
  • All services containerised with Docker Compose — fully reproducible environment

02

Stratum

Multi-tier AWS infrastructure provisioned from zero — zero console clicks.

IaC AWS ● Live

A single terraform apply provisions a complete AWS environment: VPC with public and private subnets, Internet Gateway, NAT Gateway, route tables, least-privilege security groups, and three EC2 instances across tiers. Ansible then SSHes into each server and applies role-based configuration — OS hardening, firewall rules, Nginx reverse proxy on the web tier, Java and Tomcat on the private app tier. Remote state in S3 with DynamoDB locking makes it safe for team use.

// key highlights
  • Modular Terraform — networking, compute, security, storage as separate reusable modules
  • App server has no public IP — only reachable through Nginx on the web tier
  • Ansible roles are idempotent — run ten times, result is always identical
  • All resources tagged with Project, Environment, ManagedBy, Owner

03

Vigil

Cloud-native GitOps platform — push a commit, ArgoCD deploys it automatically.

GitOps Kubernetes ● Live

A Spring Boot application running on k3s Kubernetes with full GitOps — ArgoCD watches the GitHub repository and automatically syncs every change to the cluster with zero-downtime rolling updates. Prometheus scrapes application and cluster metrics every 15 seconds. Grafana provides dashboards for cluster health, node resources, and workload performance. Custom AlertManager rules fire when pods crash, memory spikes, or the application goes down.

// key highlights
  • GitOps — Git is the single source of truth, ArgoCD self-heals any cluster drift automatically
  • Zero-downtime rolling updates: maxSurge 1, maxUnavailable 0
  • Readiness + liveness probes — traffic only routes to healthy pods
  • Custom PrometheusRules for app-specific alerting
